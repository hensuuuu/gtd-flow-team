Supabase 프로젝트 만들기
⏱ 약 3분
✅
▼

Supabase는 데이터베이스 + 로그인 + 실시간 동기화를 무료로 제공하는 서비스입니다. 기존 개인용 GTD Flow에서 이미 쓰고 있다면 같은 프로젝트를 사용해도 됩니다.

1

🔗 supabase.com 접속 → 우측 상단 "Start your project" 클릭

GitHub 계정으로 로그인하면 됩니다. GitHub 계정이 없다면 먼저 github.com에서 무료 가입.
2

"New project" 클릭 → 아래 정보 입력:

• Project name: gtd-flow (아무 이름 가능)
• Database Password: 아무 비밀번호 입력 후 반드시 메모
• Region: Northeast Asia (Tokyo) 선택 → 한국에서 빠름
• Plan: Free tier (무료)
→ "Create new project" 클릭
⚠️ 프로젝트 이름에 밑줄(_)을 쓰지 마세요! 예: gtd_flow ❌ → gtd-flow ✅
밑줄이 들어가면 나중에 Google 로그인 설정에서 오류가 납니다. 반드시 하이픈(-)을 쓰세요.
3

프로젝트 생성되면 Settings → API 메뉴로 이동

여기서 2가지를 아래 칸에 바로 붙여넣으세요. 이 가이드 전체에서 자동으로 사용됩니다.
▼ 아래 리디렉션 주소가 자동 생성됩니다
위에 URL을 붙여넣으면 여기에 자동 생성됩니다
💡 기존 GTD Flow 개인용에서 이미 Supabase를 쓰고 있다면 같은 프로젝트 URL과 key를 그대로 사용하면 됩니다. 팀용 테이블은 이름이 다르니 충돌 없어요.
2
데이터베이스 테이블 만들기 (SQL 실행)
⏱ 약 2분
✅
▼

"SQL"이라고 겁먹을 필요 없습니다. 미리 만들어둔 코드를 복사 → 붙여넣기 → 실행하면 끝입니다.

1

Supabase 대시보드 왼쪽 메뉴에서 "SQL Editor" 클릭 (아이콘: 터미널처럼 생긴 것)

2

"New query" 버튼 클릭 → 빈 편집창이 열립니다

3

아래 파일을 열어서 전체 내용을 복사합니다:

📄 supabase-team-setup.sql 열기

파일을 열면 코드가 보입니다. Ctrl+A (전체 선택) → Ctrl+C (복사)
4

Supabase SQL Editor에 Ctrl+V (붙여넣기) → 우측 하단 "Run" 버튼 클릭

초록색 "Success" 메시지가 뜨면 성공!
테이블 7개, 인덱스, 보안 정책이 모두 자동으로 만들어집니다.
⚠️ 에러가 나면? 기존에 같은 이름의 테이블이 있을 수 있습니다. "already exists" 에러는 무시해도 됩니다. 다른 에러가 나면 스크린샷 찍어서 공유해주세요.
3
Google 로그인 설정
⏱ 약 7분
✅
▼

팀원들이 Google 계정으로 로그인할 수 있게 설정합니다. Google Cloud Console에서 키를 발급받아 Supabase에 넣는 과정입니다.

1

Google Cloud Console에 접속합니다

🔗 Google Cloud Console

Google 계정으로 로그인 → 상단에 프로젝트 선택 드롭다운에서 "새 프로젝트" → 이름: GTD Flow Team → "만들기"
2

왼쪽 메뉴에서 "API 및 서비스" → "OAuth 동의 화면"

• User Type: "외부" 선택 → "만들기"
• 앱 이름: GTD Flow Team
• 사용자 지원 이메일: 본인 이메일 선택
• 개발자 연락처: 본인 이메일 입력
• 나머지는 빈칸으로 두고 "저장 후 계속" 3번 클릭해서 끝까지 진행
3

왼쪽 메뉴에서 "사용자 인증 정보" → 상단 "+ 사용자 인증 정보 만들기" → "OAuth 클라이언트 ID"

• 애플리케이션 유형: "웹 애플리케이션"
• 이름: GTD Flow Team
• 승인된 리디렉션 URI에 아래 주소를 복사해서 추가:
⬆ 1단계에서 Supabase URL을 입력하면 여기에 자동 생성됩니다
💡 이 주소는 1단계에서 입력한 Supabase URL 기반으로 자동 생성됩니다. 아직 입력 안 했다면 1단계로 돌아가서 URL을 입력해주세요.
4

"만들기" 클릭 → 팝업에서 Client ID와 Client Secret이 표시됩니다

이 두 값을 반드시 메모하세요! (닫으면 다시 볼 수 있긴 하지만 메모해두는 게 편합니다)
5

Supabase 대시보드로 돌아가서 Authentication → Providers → Google

• Enable Google provider 토글 ON
• Client ID: 방금 메모한 값 붙여넣기
• Client Secret: 방금 메모한 값 붙여넣기
• "Save" 클릭
4
Vercel에 배포하기
⏱ 약 5분
✅
▼

Vercel은 웹사이트를 무료로 인터넷에 올려주는 서비스입니다. 파일을 올리면 자동으로 주소(URL)가 생깁니다.

1

방법 A: GitHub 연결 (추천)

1. 🔗 GitHub 새 저장소 만들기
2. 저장소 이름: gtd-flow-team
3. gtd-flow-team 폴더의 3개 파일을 업로드
  → index.html, vercel.json, manifest.json
4. 🔗 Vercel 접속 → "Import Git Repository" → 방금 만든 저장소 선택 → "Deploy"

방법 B: 파일 직접 올리기 (더 간단)

1. 🔗 vercel.com 접속 → GitHub으로 로그인
2. "Add New..." → "Project"
3. 하단에 "Import Third-Party Git Repository" 대신
  → "Or, browse to upload" 를 찾아서 gtd-flow-team 폴더를 드래그&드롭
4. "Deploy" 클릭
2

배포 완료! 주소가 생성됩니다

예: https://gtd-flow-team.vercel.app

이 주소를 메모하세요 — 이게 팀원들이 접속할 앱 주소입니다.
3

확인! 3단계에서 리디렉션 URI를 이미 추가했다면 이 단계는 건너뛰세요

아직 안 했다면: Google Cloud Console → 사용자 인증 정보 → OAuth 클라이언트 편집
→ "승인된 리디렉션 URI"에 아래 주소 추가:
1단계에서 Supabase URL을 입력하면 자동 생성됩니다
💡 리디렉션 URI는 Supabase 쪽 주소입니다 (Vercel 주소가 아님). Supabase가 구글 로그인 결과를 받는 곳이에요.
5
앱에 Supabase 연결하기
⏱ 약 1분
✅
▼

마지막 단계! 배포된 앱에 접속해서 Supabase를 연결하면 끝입니다.

1

배포된 앱 주소 (예: https://gtd-flow-team.vercel.app)에 접속

2

우선 "로컬 데모 체험하기"로 기능을 먼저 둘러보세요

데모 모드에서는 로그인 없이 혼자서 칸반 보드를 써볼 수 있습니다.
3

Supabase 연결하려면: 우측 상단 ⚙️ 버튼 클릭

상단에 노란색 설정 바가 나타납니다:

• Supabase URL: 1단계에서 입력한 URL (아래에 자동 표시)
• anon public key: 1단계에서 입력한 key

→ "연결" 버튼 클릭
URL: 미입력 Key: 미입력
💡 1단계에서 입력 안 했다면 1단계로 돌아가서 Supabase URL과 Key를 입력해주세요. 앱에서 ⚙️ 누르면 같은 값을 넣게 됩니다.
4

"Google로 시작하기" 버튼을 눌러 로그인!

로그인 성공 → 워크스페이스 이름 입력 → 칸반 보드가 자동 생성됩니다.

이제 팀원들에게 이 URL을 공유하면 됩니다. 팀원들도 Google 로그인 후
초대를 받으면 같은 보드를 실시간으로 볼 수 있어요.
⚠️
문제가 생기면?
• "로그인 안 됨" → Google Cloud Console에서 OAuth 동의 화면이 "테스트" 상태인지 확인. 테스트 사용자에 팀원 이메일 추가 필요
• "redirect_uri_mismatch" → 리디렉션 URI가 정확한지 확인 (오타 주의)
• "테이블 없음" 에러 → 2단계 SQL을 다시 실행해보세요

🎉 셋업 완료!

GTD Flow Team이 배포되었습니다. 팀원들에게 URL을 공유하세요.
